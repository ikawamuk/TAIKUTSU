# TAIKUTSU
- クリック(モバイルならタップ)のみで操作できるシンプルなブラウザゲームを作る。
- ゲームは、画像をみて二者択一で正しい方を選ぶことでクリアでクリアできるまたはスコアを稼ぐという内容。
- HTML, CSS, JSのみで動かす。

## UI
- 用意する画面は、ホーム画面、ゲーム画面、ゲーム終了画面の３つ
- サイトにアクセスするとホーム画面を開く。
- 基本はスマホで操作する比率(16:9)で考える。
- 画面はすべてindex.htmlに配置し画面の切り替えはCSSの表示非表示で行う。
### ホーム画面
#### 要素
- ゲームタイトルとゲーム開始ボタンのみの最小限構成。
#### 配置
- ゲームタイトルは画面中央よりもやや上あたりに、横書きで「TAIKUTSU」。大きめに
- ゲーム開始ボタンは、画面中央よりややしたあたり。タイトルより一回り小さいサイズで、枠がある。文字は横書き「START」
#### スタイル
- 背景は白色
- タイトルの文字は水色ベースでややグラデーション。フォントはやや丸みおびていて太め。
- スタートボタンは、枠は緑色で文字が白抜きのゴシック体。
### ゲーム画面
#### 配置
- 中央に画像、その上部に問題文「退屈？」を配置。
- 画像の下に、選択肢ボタン。「退屈」「退屈じゃない」
- 左上にタイトルロゴを表示。
- 右上に正解数を表示
#### スタイル
- 問題文は黒字のゴシック体。
- 画像はフレームをつける。
- 選択肢ボタンは赤と青でどちらも角を丸くした4:6比で長方形
- タイトルロゴはタイトル画面と同じスタイルでサイズが小さいもの。
- 正解数表示は黒のゴシック体。
### 終了画面
- 最後の問題画面の上に背景透かし、スコア白文字で表示。「タイトルに戻る」「もう一度」ボタンを縦並びで表示。
- 背景は黒色で透かし。文字は透かしなしで白色

## ゲームロジック
- imageディレクトリから問題画像のセットを取得する
- 麦茶はmugicha, そうでないものはnot_mugichaディレクトリに配置する。
